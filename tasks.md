## 6. Жалобы пользователей (Word «Чат бот по репортам»)

Разбор документа с реальным прохождением куратора. Основные проблемы —
«застревание» интерфейса и запутанная навигация после генерации отчёта.

- [x] ИИ-правка периодически «зависает»: после нескольких минут ожидания
      выпадает сообщение об ошибке, а повторить действие не получается.
      → Понятные сообщения об ошибке/таймауте + гарантированный сброс busy-lock
      (`generation.py`: `_is_busy`/`_set_busy` с TTL, `finally` снимает флаг).
- [x] Кнопки «залипают» и перестают реагировать («Завершить и сформировать
      ответ», «Сохранить»), помогает только новый /start сверху чата.
      → Busy-lock проверяется ДО `cb.answer()` (иначе alert «занят» не показывался);
      TTL снимает «протухший» флаг; добавлена команда /menu как аварийный выход
      (`start.py`: `cmd_menu` чистит FSM и открывает главное меню).
- [x] После скачивания некуда вернуться — только две кнопки «PPTX/PDF».
      → После выгрузки для педагога добавлена кнопка «👦 К списку детей»
      (`export.py`: `_send_export_bottom_menu`).
- [x] Долгий путь до нужного ребёнка (7–10 кликов на цикл) и негде
      редактировать/вернуться в исходное меню при просмотре отчётов.
      → Прямой вход к меню отчёта ребёнка (просмотр/ИИ-правка/ручная правка)
      и кнопки «← Назад к меню отчёта» / «К списку детей» на всех экранах.
- [x] Отчёт помечен галочкой, но счётчик показывает «2 из 13» — непонятно,
      готов ли он.
      → Счётчик на кнопке ребёнка переформулирован: для финализированных с
      неполной анкетой показывается «готово ⚠️ N/13» (`child_menu.py`).
- [x] В PDF/PPTX несколько пунктов пустые, хотя в ИИ-правке виден текст по всем
      13 вопросам.
      → Исправлен парсер пунктов при рендере файла (пустые пункты больше не
      теряют текст).
