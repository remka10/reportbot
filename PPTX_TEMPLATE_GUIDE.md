# Руководство по настройке PPTX-шаблона

## Как добавить плейсхолдеры в report_template.pptx

Открой шаблон `Report_2026-2.pptx` в **Microsoft PowerPoint** или **LibreOffice Impress**
и замени текст в нужных текстовых блоках на плейсхолдеры ниже.

---

## Список плейсхолдеров

| Плейсхолдер | Что подставляется | Где ставить |
|---|---|---|
| `{{student_name}}` | ФИО ребёнка | Поле ФИО на обложке |
| `{{shift_name}}` | Название смены | Поле «Название смены» |
| `{{department_name}}` | Название департамента | Поле «Департамент» |
| `{{department_color}}` | **Тег перекраски** — сам текст удаляется, весь фрейм красится в цвет департамента | Поле «Департамент» (добавь к нему) |
| `{{teacher_name}}` | ФИО педагога | Поле педагога |
| `{{report_date}}` | Дата генерации дд.мм.гггг | Поле даты |
| `{{report_text}}` | Полный текст отчёта | Большое текстовое поле |
| `{{block_title}}` | Название блока | На **слайде-шаблоне блока** |
| `{{block_content}}` | Текст блока | На **слайде-шаблоне блока** |

## Как работает {{department_color}}

Если в текстовый фрейм добавить тег `{{department_color}}`:
- Код **удаляет** сам тег из текста
- Все `run`-ы (куски текста) в этом фрейме перекрашиваются в цвет департамента

**Пример:** поле «Департамент» на обложке:
```
{{department_name}}{{department_color}}
```
→ Отображается как «IT-департамент» цветом `#16A085`

## Слайд-шаблон блоков (опционально)

Если хочешь, чтобы каждый блок отчёта был на отдельном слайде:

1. Добавь в шаблон слайд с двумя текстовыми полями:
   - `{{block_title}}` — заголовок блока
   - `{{block_content}}` — текст блока
2. Этот слайд будет автоматически **дублироваться** под каждый блок отчёта
   (Блок 1, Блок 2... и т.д.)

## Быстрый старт: инспекция шаблона

```bash
# Посмотреть все shape-ы с их индексами
python scripts/prepare_pptx_template.py --inspect Report_2026-2.pptx

# Применить плейсхолдеры по карте в скрипте
python scripts/prepare_pptx_template.py
```

После запуска скрипт сохраняет результат в `app/templates/report_template.pptx`.

## Добавление нового департамента в будущем

1. Добавить запись в `DEPARTMENTS` в `app/database/models.py`
2. Добавить такую же запись в `DEPARTMENT_COLORS` и `DEPARTMENT_NAMES` в `app/services/docx_service.py`
3. Запустить миграцию: `alembic upgrade head`
